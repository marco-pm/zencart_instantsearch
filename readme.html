Introduction

This plugin uses a combination of MySQL Full-Text Search and LIKE/REGEXP queries to quickly find and sort products, brands, and categories based on their relevance to the user query.

The results can be displayed in an autocomplete dropdown as the user types in a search box and/or as a search results page with a sortable product list and infinite scroll once the user submits the search form, providing an alternative to the classic Zen Cart search results page.

Prerequisites

  • Zen Cart 1.5.7 or 1.5.8
  • PHP 7.4 through 8.2

Installation

Follow one of the following three chapters, depending on whether you're currently using Instant Search v1, Instant Search v2, or are installing Instant Search for the first time.

Upgrading from versions 1.x

  1. REMOVE all the plugin files from your site's filesystem before installing:
    • (up to version 1.0.2) searches.php
    • includes/templates/YOUR_TEMPLATE/jscript/jscript_instantSearch.js
    • includes/templates/YOUR_TEMPLATE/css/stylesheet_instantSearch.css
  2. Do a first-time install.

Upgrading from versions 2.x

Zen Cart 1.5.8

  1. REMOVE the following plugin files from your site's filesystem before installing:
    • includes/classes/ajax/zcAjaxInstantSearch.php
    • includes/languages/english/extra_definitions/YOUR_TEMPLATE/instant_search.php
    • includes/templates/YOUR_TEMPLATE/templates/tpl_ajax_instant_search_results.php
  2. Copy the plugin files to your site's filesystem.
    Rename YOUR_TEMPLATE to match the name of your site's template.
    DON'T rename the admin subdirectory under zc_plugins/InstantSearch/v3.0.0.
  3. If you're using the Bootstrap template on your store, copy the content of the bootstrap template files directory to your site's filesystem (overwrite files if asked).
  4. In the Admin, go to Modules > Plugin Manager, select Instant Search, click the Upgrade Available button and then Upgrade.
  5. In the same Admin page, click Clean Up to remove the previous plugin version files from the filesystem.
  6. In the file /includes/extra_configures/robots_pages_to_skip.php, add the line
    instant_search_result,
    under the line
    info_shopping_cart,

Zen Cart 1.5.7

  1. REMOVE the following plugin files from your site's filesystem before installing:
    • includes/classes/ajax/zcAjaxInstantSearch.php
    • includes/languages/english/extra_definitions/YOUR_TEMPLATE/instant_search.php
    • includes/templates/YOUR_TEMPLATE/templates/tpl_ajax_instant_search_results.php
  2. Write down your current Instant Search settings in the admin (Configuration > Instant Search), because they will be lost once you uninstall the current plugin.
  3. Uninstall the plugin via the Plugin Manager: in the Admin, go to Modules > Plugin Manager, select Instant Search and click the Uninstall button.
  4. Do a first-time install.
  5. Re-apply the previous Instant Search settings that you had written down on step 1.

First-time install

  1. Copy the plugin files to your site's file system.
    Rename YOUR_TEMPLATE to match the name of your site's template.
    DON'T rename the admin subdirectory under zc_plugins/InstantSearch/v3.0.0.
  2. If you're using the Bootstrap template on your store, copy the content of the bootstrap template files directory to your site's filesystem (overwrite files if asked).
  3. Install the plugin via the Plugin Manager: in the Admin, go to Modules > Plugin Manager, select Instant Search and click the Install button.
  4. Add the results listing page to the ROBOTS_PAGES_TO_SKIP value:
    • Zen Cart 1.5.8: in the file /includes/extra_configures/robots_pages_to_skip.php, add the line
      instant_search_result,
      under the line
      info_shopping_cart,
    • Zen Cart 1.5.7: in your template's version of the file /includes/languages/english/meta_tags.php, in the line that begins with define('ROBOTS_PAGES_TO_SKIP', add
      instant_search_result,
      right after
      info_shopping_cart,

Fields to search and order

With the Fields to Search and Order setting, you can decide which fields to search and in what order. The possible fields are:

  • name: product name
  • name-description: product name and description; name matches have higher priority than description matches
  • model-exact: product model, exact match (the query contains only and exactly the model)
  • model-broad: product model, broad match (the query contains also the model or part of it)
  • meta-keywords: product keywords meta tag
  • (dropdown only) category: category name
  • (dropdown only) manufacturer: manufacturer name

The fields are listed in order of search, separated by a comma. You should put the fields that you think are most relevant for your store, and/or that could provide the best matches, first. You can omit a field if you don't want to include it in the search.

For example, if the setting value is category,name-description,model-exact then:

  1. First, the category names are searched, the matches are sorted by relevancy, and then added on top of the results.
  2. If the maximum number of results to display has not been reached, the product names and descriptions are searched, the matches are sorted by relevancy, and then added at the end of the current results.
  3. If the maximum number of results to display has not been reached, the product models are searched, the matches are sorted by relevancy, and then added at the end of the current results.

You must pick only one field between name and name-description. If you don't want to include the product description in the search, or the search is too slow/inconsistent, use name.

The meta-keywords field refers to the keywords meta tag field that you can set per-product via the Meta Tags Editor. This is useful if you want to add some keywords that are not present in the product's name/description but you still want to associate with the product in the search.
Note: Meta keywords are no longer used by search engines (Google & co.) since a long time ago, so don't be afraid to affect your website's SEO ranking by editing that field. Any value it contains is ignored by search engines.

Query Expansion

Enable Query Expansion to show also related products in the results.

Suppose you have three products with names:

  • Microsoft Mouse and Keyboard set
  • Logitech Advanced Mouse
  • Optical wireless mouse by Trust

The user searches logitech.
If Query Expansion is disabled, only the second product (the one with logitech in the name) will be shown.
If Query Expansion is enabled, the other two products will also be shown (with less priority), because they contain the word mouse, which is present in the matched product, so they could be relevant for the user.

Query Expansion also applies to the product description (if included in the fields list).

Note: this option is not equivalent to a typo-tolerance or synonym support feature (e.g. Did you mean...?). A search with the keyword logiech (typo), or logitechs, or mice would return no matches.

FAQs & Troubleshooting

What's the difference between Instant Search and the Advanced Search included in Zen Cart?

Instant Search uses Ajax (no page reload) and searches for matches only on specific fields (product's name, model and description) with the goal of being fast and relevant. It does not load all results at once, but rather stops once it has found enough, and sorts them by relevance to the user query. It can also find similar results (Query Expansion).

The storefront (formerly "Advanced") Search doesn't use Ajax and aims for completeness and flexibility. The user can apply various filter conditions, such as limiting the search to certain categories or price ranges and using AND/OR in the query. It is therefore the tool to use for tailored searches on all available fields. The storefront Search also shows the exact total number of results, but in order to do this and provide pagination, it must first retrieve them all from the database. Additionally, it does not sort the results by relevance.

What's the difference between Instant Search and the Ajax Search of the Bootstrap template?

Aside from a visual difference (Bootstrap Ajax search shows results in a modal instead of a dropdown), both use Ajax (no page reload) and only look for matches on specific product fields (product's name and model, also product's description and meta keywords – if enabled – in the case of Instant Search).
The difference is that Bootstrap Ajax Search is still partially based on the storefront Search algorithm. It retrieves all the results first to count them, and it doesn't sort them by relevance. Compared to the storefront Search though, it's very fast.
Depending on your needs, if you're using the Bootstrap template you may prefer the Bootstrap Ajax Search to Instant Search. You can easily switch between them in the configuration settings.

The search is slow

Ideally, the results should be returned in less than 1 second.
If that's not the case, disable the search in product descriptions (i.e. in the admin Fields to Search and Order setting, use name instead of name-description).
If the search is still slow, disable Query Expansion.

The autocomplete dropdown does not appear

If you have enabled dropdown results and nothing happens when the user types, you may be using a custom template and your search entry boxes have a different CSS selector. You can set the CSS selector in the admin Dropdown - Input Box Selector setting.

The look of the results listing page doesn't fit my store template

The plugin provides template files for Responsive Classic and Bootstrap templates. If you're using a different template, you might need to customize these files to fit your store's theme:

  1. /includes/modules/YOUR_TEMPLATE/product_listing_instant_search.php
  2. /includes/templates/YOUR_TEMPLATE/templates/tpl_instant_search_result_default.php
  3. /includes/templates/YOUR_TEMPLATE/templates/tpl_ajax_instant_search_results_listing.php

The search returns too many non-relevant results

Disable Query Expansion.

The results listing page is slow when a sort is applied

A custom sort (e.g. sort by price) in the product list basically negates the speed advantage of Instant Search, because it has to find and load all products first in order to sort them.

Does the search include typo-tolerance and synonym support (e.g. Did you mean...?)

Not yet (work in progress!).
